# Qu'est-ce que llms.txt ?

[llms.txt](https://llmstxt.org/) est une proposition de standart pour aider les ia a comprendre le contenu d'un site web.

Au lieu d'essayer de parser des pages html, le standard propose l'ajout d'un fichier `llms.txt` listant toutes les pages accessibles, un peu comme le `sitemap.xml` fait déjà, mais en y ajoutant des sections et descriptions pour chaque lien.\
Ainsi que l'ajout d'une version markdown de chaque page, ainsi en allant sur la page [https://vahor.fr/project/llms-txt](https://vahor.fr/project/llms-txt) on a une belle version html avec du style et du js, mais en allant sur [https://vahor.fr/project/llms-txt.md](https://vahor.fr/project/llms-txt.md) on peut voir tout ce contenu en format Markdown, beaucoup plus lisible pour une IA ou n'import quel autre outil de scraping.

# Pouquoi un outil pour générer ces fichiers ?

Le deuxieme est plus simple et n'a pris que 4s a générer le fichier `llms.txt`.

## Alors pourquoi en faire un de plus ?

L'inconvenient de ces deux outils est qu'ils ne génèrent que des fichiers `llms.txt` et `llms-full.txt`, ils ne génèrent pas les fichiers `.md` de vos pages.\
De plus ils ne sont pas très customisables.

<UrlPreview url="https://github.com/Vahor/llms-txt" />

Ce que je cherche c'est:
- Pourquoi modifier les titres de pages, par exemple retirer les ` | Vahor`.
- Nommer les sections et grouper les liens comme je le veux.
- Modifier les descriptions si besoin.
- Générer des fichiers `.md` pour chaque page.
- Pouvoir transformer le markdown avant de l'exporter.
   - Très souvent on utilise [remark](https://github.com/remarkjs/remark) pour changer le markdown brut en contenu enrichi : par exemple sur [vahor.fr](https://vahor.fr), les liens d'images ou autres sont relatifs au fichier markdown par exemple `../../../public/fichier.txt`. Le garder tel quel rend n'est pas bon, on veut retirer ce prefix avant de publier.
   - Retirer des champs dans la partie frontmatter. Par exemple certaines fois on ajoute des tags, une configuration de sidebar etc. Mais on souhaite peut être ne concerver que le titre et la description.
   - Ou toute autre chose qui modifie le markdown avant de l'exporter.

# Comment ça marche ?

Une petite dépendance en plus dans votre projet node :

```bash
bun add -D @vahor/llms-txt
```

Puis on ajoute un fichier ts qui sera exécuté lors de la génération de votre site :

```ts title="scripts/generate-llms.ts"
import {
	LLMS_TXT_FILENAME,
	type PluginOptions,
	generate,
} from "@vahor/llms-txt";
import { allDocuments } from "contentlayer/generated";
import { visit } from "unist-util-visit";

// Ici on retire les ../../ pour les images
function fixRelativeLinks() {
	return (tree: any) => {
		const pattern = /(..\/)*public\//g;
		visit(tree, "image", (node) => {
			node.url = (node.url as string).replace(pattern, "/");
		});
		visit(tree, "html", (node) => {
			node.value = (node.value as string).replace(pattern, "/");
		});
		visit(tree, "code", (node) => {
			node.value = (node.value as string).replace(pattern, "/");
		});

		return tree;
	};
}

const options = {
	outputPath: (path) => {
		if (path === LLMS_TXT_FILENAME) {
			// Là on dit où mettre le fichier `llms.txt`
			return "./public/llms.txt";
		}
		const slug = path.split("/").slice(3).join("/");
		const withoutExtension = slug.split(".").slice(0, -1).join(".");
		// Ici on dit où mettre le fichier `.md` de chaque page
		return `./public/${withoutExtension}.md`;
	},
	formatFrontmatter: (frontmatter) => ({
		// On ne conserve que le titre, la description et la date de publication
		title: frontmatter.title,
		description: frontmatter.description,
		datePublished: frontmatter.datePublished,
	}),
	sections: [
		{
			// Ici on met le titre qu'on souhaite pour la section
			title: "Vahor.fr",
		},
		{
			title: "Posts",
			// Et ici on liste tous les liens qu'on souhaite dans la section "Posts".
			links: allDocuments.map((doc) => ({
				title: doc.title,
				url: `https://vahor.fr/${doc.pageType}/${doc.slug}.md`,
				description: doc.description,
			})),
		},
	],
	content: allDocuments.map((doc) => ({
		path: `./content/${doc._raw.sourceFilePath}`,
	})),
	remarkPlugins: [fixRelativeLinks],
} satisfies PluginOptions;

generate(options);
```

Plus qu'à lancer un `bun ./scripts/generate-llms.txt.ts` avant de publier le site. Et le script va se charger de générer le fichier `llms.txt` et une version `.md` de chaque page.
